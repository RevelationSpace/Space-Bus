Detailed Description

An SBLP hardware-side implementation for the ATTiny85.

This file implements the hardware-dependent parts of the SBLP protocol. It is specific to the Atmel ATTiny85 microcontroller.

ISR ( USI_OVF_vect  )

USI overflow vector - process the transmitted or received byte This ISR gets run whenever the USI overflows - for us, this means a byte has been transmitted or received. As we're abusing the USI as a hacky UART, we may need to send another byte. If not, notify the layer above via the appropriate struct hw_callbacks members.

void t485_begin_transmission ( )

Start a transmission. This will tell the line driver to take the bus off high-impedance mode so other nodes can see our transmission. It also turns the pin-change interrupt off so we don't get distracted by seeing our own data.

void t485_end_transmission ( )

End a transmission. This will return the bus to high-impedance mode and turn the pin- change interrupt back on.
